========================================
  KEY FINDINGS FOR FIX IMPLEMENTATION
========================================

1. ip route get 8.8.8.8 → dev ccmni1
   Pattern: ccmni* = Cellular (MediaTek)
   This is the PRIMARY reliable source.

2. /proc/net/route default-route parser is BROKEN
   The default route is NOT destination=00000000 on this device.
   It uses destination=0000000A (10.0.0.0) with mask=000000FF.
   Fix: look for Flags=0001 (RTF_UP) on the main data interface.

3. dumpsys connectivity "Active default network:" WORKS on Android 13
   Format: "Active default network: <id>" then NAI block with:
   - ni{MOBILE[NR] CONNECTED ...}
   - lp{{InterfaceName: ccmni1 ...}}
   - nc{[ Transports: CELLULAR ...]}
   Parser should extract: (a) interface name from lp, (b) transport from nc.

4. dumpsys wifi is definitive for WiFi OFF state
   WifiState=0 + curState=DisabledState = WiFi disabled
   Don't rely on cached SSID from connection history.

5. Dual-SIM device shows multiple ccmni* interfaces
   ccmni0 = IMS (VoLTE) - metered=no, no internet
   ccmni1 = default data - internet, metered=?
   ccmni2 = secondary IMS
   The default route points to ccmni1 specifically.

6. wlan0 has RX/TX counters even when WiFi is OFF
   Counters are stale (869KB/613KB). Don't use /proc/net/dev
   counters to infer active network.
