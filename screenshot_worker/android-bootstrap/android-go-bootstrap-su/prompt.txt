create a golang static binary for android, (must with su or root permission), run this binary as service without asking for mediaprojection permission per session, check if root available or not, if not, ask for root permission.

refer to android-reference/flashback-android/flashback-termux/flashback/workers/screenshot.py and files related to it to create the golang binary

write the golang code to android-go-bootstrap-su

commandline syntax is like go-bootstrap

golang binary will periodically output service status according to config file, like 3s, with a fixed format in json, prefix or keyword, e.g: SERVICE_BRIEFING {"status": "running"} or {"status": "stopped"} and many more attributes, so that we can capture, parse and display it in diagnostic page.
